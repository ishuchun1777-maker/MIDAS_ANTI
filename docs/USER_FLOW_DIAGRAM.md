# MIDAS User Flow Diagram

This document describes the core user flows of the MIDAS platform.

MIDAS is an advertising marketplace connecting:

- Buyers (entrepreneurs who want advertising)
- Sellers (media owners and influencers)
- Specialists (marketers, designers, media buyers)

The platform must guide users through simple and clear workflows.

---

# Core Marketplace Flow

Seller creates advertising asset

→ Buyer discovers asset

→ Buyer sends offer

→ Seller accepts offer

→ Deal is created

→ Seller delivers advertising

→ Buyer approves

→ Both users rate each other

---

# Buyer Flow

Step 1: Register

User registers using:

Telegram login  
Email login  

---

Step 2: Select Role

User selects role:

Buyer

---

Step 3: Create Buyer Profile

User fills:

Business name  
Industry  
Target audience  
Budget range  

---

Step 4: Buyer Dashboard

Buyer lands on dashboard.

Dashboard shows:

Active campaigns  
Offers  
Deals  
Recommendations  

---

Step 5: Discover Advertising

Buyer opens discovery page.

Filters available:

Media type  
City  
Audience  
Price range  

---

Step 6: View Asset

Buyer opens asset detail page.

Shows:

Seller profile  
Audience information  
Pricing  
Portfolio  

---

Step 7: Send Offer

Buyer sends offer.

Offer includes:

Price  
Message  
Campaign brief  

---

Step 8: Offer Status

Offer status:

Pending  
Accepted  
Rejected  
Countered  

---

Step 9: Deal Creation

If seller accepts:

Deal room is created.

---

Step 10: Deal Room

Includes:

Chat  
Deliverables  
Timeline  
Status  

---

Step 11: Deliverable Review

Seller uploads proof.

Buyer can:

Approve  
Request revision  

---

Step 12: Completion

Deal status becomes:

Completed

---

Step 13: Rating

Buyer rates seller.

---
